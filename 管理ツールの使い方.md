# ニラトルフSTN 管理ツール (AdminTool) の使い方

このドキュメントでは、各プレイヤー（PL）のアクセス権限を簡単に操作・変更するためのローカル管理ツール `AdminTool.bat` の使い方について解説します。

## 1. 概要
`AdminTool.bat` は、ウェブサイトのアクセス制御ファイルである `setting.json` を安全に書き換え、さらにGitHubへその変更を自動的にプッシュ（反映）するためのツールです。
このツールは**キーパー（管理者）のパソコン（ローカル環境）でのみ動作**し、プレイヤーに見られることはありません。

## 2. 起動方法
1. GitHubのローカルフォルダ（`Nilatorf` フォルダ）を開きます。
2. **`AdminTool.bat`** をダブルクリックして起動します。
3. コマンドプロンプト（黒または青い画面）が立ち上がり、英語のメニューが表示されます。

## 3. メニューと基本操作
ツールを起動すると、現在の `setting.json` の設定状態が表示された後に、メニューが表示されます。キーボードの数字を入力して `Enter` を押して操作します。

- **`1`**: 指定したPLのステータスを変更・または新規追加します。
- **`2`**: 現在登録されているすべてのPLのステータスを、一括で同じものに変更します。
- **`9`**: 行った変更をGitHubへ保存（Push）し、実際のウェブサイトに反映させます。
- **`0`**: ツールを終了します。

## 4. 複数PLの同時変更・追加機能について (メニュー1)
メニューで `1` を選ぶと、変更したいPLの番号（数字）を尋ねられます。

### ① 単一のPLを変更する場合
`1` や `5` などのように数字を1つだけ入力します。

### ② 複数のPLを同時に変更する場合（カンマ区切り）
`1,2,5` のように、数字をカンマで区切って入力すると、それら複数のPLを一気に同じステータスに変更することができます。

### ③ 新しいPLを追加する場合
まだ設定ファイルに存在しない新しい番号（例: `12,13,14`）を入力した場合、**自動的にそのIDが新規作成**され、指定したステータスが割り当てられます。参加人数が増えた場合もこの方法で簡単に追加できます。

## 5. ステータスの種類
PLを指定した後に、以下の3つから新しい状態を選びます。

- **`1` (ok)**: 正常なアクセスを許可し、ポータル画面を表示させます。
- **`2` (ng)**: システムエラー画面を表示させ、フリーズ状態にします。
- **`3` (t)**: Boothの外部ページへ強制的に転送します。

## 6. GitHubへの反映 (メニュー9)
設定を変更しただけでは、実際のウェブサイト（オンライン）には反映されません。
変更が終わったら、必ず最初のメニューで **`9`** を入力してください。

自動的に `git pull --rebase` （最新状態の取り込み）が行われた後、`git push` が実行され、数分以内にウェブサイト上のアクセス制御が更新されます。

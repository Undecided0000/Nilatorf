# ニラトルフSTN ウェブサイトシステム解説

このドキュメントでは、「ニラトルフSTN (students and Teachers Navigator)」ウェブサイトのアクセス制御とシステムの仕組みについて解説します。

## 1. アクセス制御の基本方針
当ウェブサイトは、プレイヤー(PL)に渡されたNFCタグ等からの直接アクセスのみを許可し、それ以外のアクセスや、権限のないPLのアクセスをブロックまたは外部サイトへ転送する仕組みを採用しています。
アクセス制御はサーバー側ではなく、**フロントエンド（JavaScript）による動的な判定**で行われています。

## 2. URLパラメーターによる判定
アクセスURLの末尾に付与されるパラメーターによって、サイトの表示内容が変化します。
例： `https://undecided0000.github.io/Nilatorf/?src=nfc&id=PL1`

- **`src=nfc`**
  このパラメーターが存在しない場合、直リンク（URLの手入力など）と判定され、画面には何も表示されません（真っ白な画面になります）。
- **`id=PL1`**
  アクセスしてきたプレイヤーのIDを指定します。このIDに基づいて、後述の `setting.json` からステータスを読み込みます。

## 3. `setting.json` によるステータス管理
各プレイヤー（PL）がどの画面を見ることができるかは、ルートディレクトリにある `setting.json` ファイルで一元管理されています。

```json
{
  "PL1": "ok",
  "PL2": "ng",
  "PL3": "t"
}
```

### ステータスの種類と挙動
- **`ok` (通常アクセス許可)**
  正常なポータル画面を表示し、学生データの同期や教科書ダウンロードなどの操作が可能になります。
- **`ng` (エラー画面フリーズ)**
  「SYSTEM ERROR」という警告ウィンドウが画面中央に表示され、一切の操作を受け付けないフリーズ状態になります。
- **`t` または 未定義 (転送)**
  このステータスが指定されている、あるいは `setting.json` にそのPLの記載が存在しない場合、自動的にBoothの指定ページ（販売ページ等）へリダイレクト（転送）されます。デフォルトの動作はこれになります。

## 4. 内部の動作フロー (`index.html`)
1. ユーザーがURLにアクセスすると、`index.html` が読み込まれます。
2. ページロード時 (`DOMContentLoaded`) にJavaScriptがURLパラメーターを解析します。
3. `src=nfc` がない場合はすぐに処理を停止し、画面を空にします。
4. `setting.json` を非同期（`fetch`）で読み込み、URLパラメーターの `id` と照らし合わせます。
5. 照合されたステータス (`ok`, `ng`, `t`) に従って、該当するHTML要素（正常画面やエラー画面）を表示、または別ページへ転送します。
6. `ok` の場合は続けて、対象PLの画像データ読み込みや、テキストファイル教材の動的生成処理が行われます。
